# Rick's Basement Frontend

Use when editing the Next.js app, React components, Tailwind styling, PixiJS animations, Zustand stores, or basement UI/UX.

Rules:

- Preserve the 8-bit animated basement identity.
- Keep the UI game-like, readable, and operational.
- Use original procedural pixel characters only; do not add copyrighted sprites or likenesses.
- Use normalized event states from `frontend/src/lib/eventTypes.ts`.
- Do not add LLM calls or prompt summarization.
- Verify state transitions with Vitest when changing reducers.
