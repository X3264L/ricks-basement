- Rick's Basement must remain local-first.
- Do not add cloud requirements for MVP.
- Do not add LLM calls inside hooks.
- Do not store full prompts by default.
- Do not store full command outputs by default.
- Do not store secrets.
- Do not print natural language from hooks.
- Do not create office metaphors.
- Do not copy another project's theme.
- Keep the sci-fi basement identity strong.
- Prefer minimal telemetry.
- Every feature must preserve token efficiency.
- Add tests for privacy redaction.
- Add tests for event normalization.
- Keep modules small.
- Keep setup simple.
- Treat README quality as part of the product.
